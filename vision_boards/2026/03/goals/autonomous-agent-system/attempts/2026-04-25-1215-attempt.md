## Step 1: Create a Telegram Bot with BotFather

1. **Open Telegram** and search for `@BotFather` (or visit https://telegram.me/BotFather)
2. **Start the bot** by clicking "Start" or sending `/start`
3. **Create a new bot** by sending `/newbot` or clicking the menu button
4. **Follow the prompts**:
   - Enter a **display name** for your bot (e.g., "Goal Agent")
   - Enter a **username** ending in `_bot` (e.g., `MyGoalAgent_bot`)
   
5. **Copy the bot token** — BotFather will send a message like:

```
Done! Congratulations on your new bot. You will find it at t.me/MyGoalAgent_bot.

Use this token to access the HTTP API:
1234567890:ABCdefGHIjklMNOpqrsTUVwxyz123456789
```

⚠️ **Keep this token secure!** It grants access to control your bot.

---

## Step 2: Get Your Chat ID

### Method A: Using @userinfobot (Easiest)

1. **Search for `@userinfobot`** in Telegram (or visit https://telegram.me/userinfobot)
2. **Start the bot** by clicking "Start"
3. The bot will display your user information including your **Chat ID** (a number like `123456789`)

### Method B: Using getUpdates API

If Method A doesn't work in your region:

1. **Open Telegram** and search for your new bot (e.g., `@MyGoalAgent_bot`)
2. **Start the bot** by sending `/start` (this is important!)
3. **Open your browser** and visit:
   ```
   https://api.telegram.org/bot<YOUR_BOT_TOKEN>/getUpdates
   ```
   Example: `https://api.telegram.org/bot1234567890:ABCdefGHIjklMNOpqrsTUVwxyz123456789/getUpdates`
4. **Look for `"id"`** in the JSON response under `message.chat`:
   ```json
   {
     "ok": true,
     "result": [{
       "message": {
         "chat": {
           "id": 123456789,
           "type": "private"
         }
       }
     }]
   }
   ```
   The number `123456789` is your Chat ID.

### Method C: Using @RawDataBot

1. **Forward any message** from your bot chat to `@RawDataBot` (https://telegram.me/RawDataBot)
2. The bot will display raw data including your **Chat ID**

## How Clarification Questions Work

When the autonomous agent encounters a task requiring clarification:

1. **Agent identifies need** — The LLM determines it can't proceed without more information
2. **Message is sent** — The `telegram.py` module sends a message like:

```
🤔 Clarification needed for task: "Research immigration requirements for Tennessee"

I need to know: What type of visa are you currently on in Mexico?
```

3. **Wait for response** — The run ends with the task marked as "blocked"
4. **User responds** — You reply via Telegram to the same bot chat
5. **Next run picks up** — The next agent run reads your response from the attempt file and continues

## Troubleshooting

### "result": [] (Empty Updates)

**Problem:** getUpdates returns empty results

**Solution:** 
1. Ensure you've sent `/start` to your bot first
2. Disable privacy mode (Step 4)

### Bot Not Responding

1. **Check token format** — Should be `numbers:letters`
2. **Verify Chat ID** — Make sure it's correct for your private chat
3. **Check API URL** — Use `api.telegram.org/bot<token>/getUpdates`

### Messages Not Delivered

1. **Verify Chat ID** is correct (positive number for private chats)
2. **Check bot permissions** — Bot must have permission to message you
3. **Try sendMessage API** in browser first

## Appendix: Telegram Bot API Quick Reference

### Send Message

```python
import requests

def send_telegram_message(token: str, chat_id: str, text: str):
    url = f"https://api.telegram.org/bot{token}/sendMessage"
    data = {
        "chat_id": chat_id,
        "text": text,
        "parse_mode": "Markdown"
    }
    response = requests.post(url, json=data)
    return response.json()
```

### Available Message Types

| Method | Purpose |
|--------|---------|
| `sendMessage` | Text messages |
| `sendPhoto` | Images |
| `sendDocument` | Files |
| `sendLocation` | GPS coordinates |
| `sendPoll` | Create polls |

### Parse Modes

- `Markdown` — Legacy markdown formatting
- `MarkdownV2` — Modern markdown (recommended)
- `HTML` — HTML formatting
