# Telegram Bot Setup Guide for Goal Agent

## Overview

The Telegram bot serves two purposes in the autonomous goal execution system:
1. **Clarification Questions** — When the agent is blocked on a task and needs user input
2. **Run Summaries** — After each execution run, the agent reports progress

This guide covers the complete setup from creating the bot to testing the integration.

---

## Step 1: Create a Telegram Bot via BotFather

### Process

1. **Open Telegram** and search for `@BotFather`
2. **Start the conversation** by clicking "Start" or sending `/start`
3. **Create a new bot** by sending `/newbot`
4. **Choose a name** for your bot (e.g., "GoalAgentBot" or "VisionBoardAssistant")
5. **Choose a username** — Must end in `bot` and be unique (e.g., `MyGoalAgentBot`)
6. **Save the API token** — BotFather will return a token like `123456:ABC-DEF1234ghIkl-zyx57W2v1u123ew11`

### Important Notes

- **Keep the token secret** — Anyone with the token can control your bot
- **You can always get the token again** — Message `@BotFather` and send `/token` to see your bots
- **You can regenerate the token** — Send `/revoke` to invalidate the current token and get a new one

## Step 2: Get Your Telegram Chat ID

### Method 1: Using userinfobot (Recommended)

1. **Search for `@userinfobot`** in Telegram
2. **Start the bot** by clicking "Start"
3. **Your Chat ID will be displayed** — It looks like a large number (e.g., `123456789`)

### Method 2: Using @getidsbot

1. **Search for `@getidsbot`** in Telegram
2. **Start the bot** and send any message
3. **Find your ID in the response**

### Method 3: Using Telegram Web

1. Go to [web.telegram.org](https://web.telegram.org)
2. Log in to your account
3. Open any chat
4. Look at the URL — The chat ID is the number after `#/im?p=` — but this only works for groups

### Notes

- **Your Chat ID is a number** — Not the same as your username
- **Group chats have negative IDs** — E.g., `-123456789`
- **For the goal agent** — Use your personal Chat ID (positive number)

## Step 4: Test the Telegram Bot

### Quick Test Script

Create a file `test_telegram.py`:

```python
#!/usr/bin/env python3
"""Quick test to verify Telegram bot credentials."""

import os
import json
import urllib.request
import urllib.error

def send_message(text: str) -> dict:
    """Send a test message via Telegram Bot API."""
    token = os.environ.get("TELEGRAM_BOT_TOKEN")
    chat_id = os.environ.get("TELEGRAM_CHAT_ID")
    
    if not token or not chat_id:
        print("ERROR: TELEGRAM_BOT_TOKEN and TELEGRAM_CHAT_ID must be set")
        return {"ok": False, "error": "Missing credentials"}
    
    url = f"https://api.telegram.org/bot{token}/sendMessage"
    data = json.dumps({
        "chat_id": chat_id,
        "text": text,
        "parse_mode": "Markdown"
    }).encode()
    
    req = urllib.request.Request(
        url, 
        data=data,
        headers={"Content-Type": "application/json"}
    )
    
    try:
        with urllib.request.urlopen(req) as resp:
            return json.loads(resp.read().decode())
    except urllib.error.HTTPError as e:
        error_body = e.read().decode()
        print(f"HTTP Error {e.code}: {error_body}")
        return {"ok": False, "error_code": e.code, "error": error_body}

if __name__ == "__main__":
    print("Testing Telegram Bot credentials...")
    result = send_message("🔔 *Goal Agent Test*\n\nCredentials configured successfully! ✅")
    
    if result.get("ok"):
        print("✅ Test passed! Bot is working.")
    else:
        print(f"❌ Test failed: {result}")
```

## Step 5: Verify Existing Integration

The `telegram.py` module should already handle the messaging. Here's how it works:

### Current Implementation Pattern

```python
# scripts/goal-agent/telegram.py

import os
import json
import urllib.request
import urllib.error

class TelegramClient:
    """Client for sending messages via Telegram Bot API."""
    
    def __init__(self):
        self.token = os.environ.get("TELEGRAM_BOT_TOKEN")
        self.chat_id = os.environ.get("TELEGRAM_CHAT_ID")
        self.api_base = f"https://api.telegram.org/bot{self.token}" if self.token else None
    
    def is_configured(self) -> bool:
        """Check if credentials are configured."""
        return bool(self.token and self.chat_id)
    
    def send_message(self, text: str, parse_mode: str = "Markdown") -> dict:
        """Send a message to the configured chat."""
        if not self.is_configured():
            return {"ok": False, "error": "Telegram not configured"}
        
        url = f"{self.api_base}/sendMessage"
        data = json.dumps({
            "chat_id": self.chat_id,
            "text": text,
            "parse_mode": parse_mode
        }).encode()
        
        req = urllib.request.Request(
            url,
            data=data,
            headers={"Content-Type": "application/json"}
        )
        
        try:
            with urllib.request.urlopen(req) as resp:
                return json.loads(resp.read().decode())
        except urllib.error.HTTPError as e:
            error_body = e.read().decode()
            return {"ok": False, "error_code": e.code, "description": error_body}
    
    def send_clarification_question(self, task_title: str, question: str) -> dict:
        """Send a clarification question for a blocked task."""
        message = (
            f"⚠️ *Clarification Needed*\n\n"
            f"*Task:* {task_title}\n\n"
            f"{question}\n\n"
            f"*Please reply with your answer.*"
        )
        return self.send_message(message)
    
    def send_run_summary(self, tasks_processed: int, tasks_successful: int, 
                       tasks_blocked: int, message: str) -> dict:
        """Send a run summary after goal agent execution."""
        status_emoji = "✅" if tasks_successful > 0 else "⚪"
        message_text = (
            f"{status_emoji} *Goal Agent Run Summary*\n\n"
            f"*Processed:* {tasks_processed} tasks\n"
            f"*Successful:* {tasks_successful}\n"
            f"*Blocked:* {tasks_blocked}\n\n"
            f"{message}"
        )
        return self.send_message(message_text)
```

### Expected Message Types

1. **Clarification Questions** — When the agent encounters ambiguous instructions
2. **Run Summaries** — After each execution cycle (every 2 hours)

## Troubleshooting

### Common Issues

| Issue | Cause | Solution |
|-------|-------|----------|
| `401 Unauthorized` | Invalid bot token | Re-check token from `@BotFather` using `/token` |
| `400 Bad Request` | Invalid chat ID | Verify with `@userinfobot` |
| `403 Forbidden` | Bot blocked by user | User must start a chat with the bot first |
| `429 Too Many Requests` | Rate limiting | Wait 30 seconds, implement retry logic |

### Verify Bot Started

The user **must** start a conversation with the bot first:
1. Open the bot in Telegram
2. Click "Start" or send `/start`
3. The bot can then send messages to that user

## Security Best Practices

1. **Never commit the bot token** — Always use GitHub secrets
2. **Rotate tokens periodically** — Use `/revoke` in BotFather
3. **Limit bot permissions** — Only grant what's needed (send messages)
4. **Don't share the bot** — Create separate bots for different purposes
5. **Monitor usage** — Check for unusual activity
